🎓 AI 튜토리얼: Few-shot으로 AI 전문가 만들기

💡 Logic (원리)

Few-shot Learning은 AI에게 구체적인 예시를 보여주는 기법입니다. Ep 1의 Role Play가 "역할 부여"였다면, Few-shot은 "사례 학습"입니다.

핵심 원리:

1. 예시 = 맥락: AI는 예시를 보고 패턴을 학습합니다.

2. 1-shot vs Few-shot: 예시 1개 = 1-shot, 2개 이상 = Few-shot

3. 품질 > 수량: 예시 100개보다 정확한 예시 3개가 더 효과적입니다.

왜 Few-shot이 필요한가?

- Role Play만으로는 조직 고유의 규칙/용어를 학습 불가

- 실제 사례를 보여줘야 정확한 답변 생성

- 윤서연처럼 "구산구 ≠ 대진시" 문제 해결

🛠️ Stack (도구)

- **ChatGPT** (무료/유료)

- 조직 내 사례집, 매뉴얼, 과거 처리 사례

- 엑셀/노션에 정리하면 더 좋음

📋 Guide (실행 단계)

Step 1: 사례 수집 (3~5개 추천)

✅ 좋은 사례: - 실제 발생한 사건 - 명확한 입력/출력 쌍 - 조직 고유 용어 포함 ❌ 나쁜 사례: - 가상의 예시 - 애매한 설명 - 일반론적 내용

Step 2: 사례를 프롬프트에 삽입

구조:

[Role Play] + [Few-shot 예시 1] [Few-shot 예시 2] [Few-shot 예시 3] + [내 질문]

Step 3: 테스트 & 개선

- 답변이 부정확하면 예시 추가

- 예시가 너무 많으면 핵심만 남기기

- 조직 변화 시 예시 업데이트

🎯 Prompt (바로 쓰는 템플릿)

기본 템플릿:

너는 [조직명] [부서]의 [직무] 담당자야. 아래는 우리 조직의 실제 처리 사례야: 사례 1: - 상황: [구체적 상황] - 처리: [실제 처리 방법] - 결과: [최종 결과] 사례 2: - 상황: [구체적 상황] - 처리: [실제 처리 방법] - 결과: [최종 결과] 사례 3: - 상황: [구체적 상황] - 처리: [실제 처리 방법] - 결과: [최종 결과] 이제 아래 상황을 위 사례들과 같은 방식으로 처리해줘: [내 질문]

실전 활용 예시 (대진시 버전):

너는 대진시 민원봉사과 담당자야. 아래는 대진시 민원 처리 실제 사례야: 사례 1: - 상황: 베트남 배우자 출생신고 (한글 서류 없음) - 처리: 대진시 조례 제3조 → 번역 공증본 + 아포스티유 - 결과: 3일 내 처리 완료 사례 2: - 상황: 전입신고 시 주소 오기재 - 처리: 대진시 조례 제7조 → 정정신청서 + 주민등록등본 - 결과: 당일 처리 사례 3: - 상황: 주차 위반 이의신청 (주민 항의) - 처리: 대진시 조례 제12조 → CCTV 증거 확인 → 거부 - 결과: 법적 근거 제시로 민원 종결 이제 아래 상황을 위 사례처럼 처리해줘: 외국인 유학생이 비자 연장 서류를 영어로 제출했습니다. 어떻게 처리하나요?

Tips:

- 사례는 3~5개가 최적 (너무 많으면 혼란)

- 각 사례에 조직 고유 조례/규정 번호 명시

- "위 사례처럼"이라는 지시어로 패턴 강조
