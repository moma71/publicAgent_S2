낯선 목소리였다. 빠르고 논리적이었다.

"행정안전부 AI혁신팀 최지훈 사무관입니다."

윤서연은 안경을 밀어 올렸다. 손가락이 책상을 두드렸다. 또다닥.

"네, 말씀하세요."

"대진시 민원 처리 시스템이 화제더라고요. 직접 뵙고 싶은데, 시간 괜찮으세요?"

"...네?"

오전 11시 03분.

최지훈은 노트북을 돌렸다. 화면 가득 그래프.

윤서연은 화면을 봤다. 복잡한 시스템 구조도.

"이론상 완벽합니다. 중앙에서 표준화하면 모든 지자체가 동일한 수준의 서비스를 제공할 수 있죠."

최지훈은 또박또박 말했다. 손가락으로 하나씩 펴가며.

"첫째, 데이터 표준화. 둘째, AI 모델 공유. 셋째, 예산 절감."

"하지만..."

윤서연은 입술을 깨물었다.

"네?"

"이거... 대진시랑 안 맞을 것 같은데요."

정적.

"...왜죠?"

"조례가 달라서요. 각 지자체마다."

최지훈은 멈췄다.

오전 11시 47분.

"민준 선배님도 오셨어요?"

윤서연은 놀랐다.

강민준은 웃었다. 안경을 밀어 올렸다.

"서연아, 잘 지냈지?"

"네, 선배님."

"서연 주무관, 구체적으로 뭐가 문제죠?"

윤서연은 화이트보드에 적었다.

서울시 조례 제5조: 외국인 배우자 출생신고 - 번역본만 필요 대진시 조례 제3조: 외국인 배우자 출생신고 - 번역본 + 아포스티유 필요

"아포스티유?"

"네. 대진시는 추가 서류를 요구해요. 230개 지자체 조례가 다 달라요."

최지훈은 노트북을 열었다. 타닥타닥타닥.

"...맞네요. 데이터베이스를 확인해보니 정말 다르군요."

민준은 팔짱을 꼈다.

오후 1시 22분.

"RAG요?"

윤서연은 펜을 들었다.

"Retrieval-Augmented Generation. 검색 증강 생성."

최지훈은 화이트보드에 그렸다.

[Step 1] 대진시 조례 200페이지를 AI에 학습 [Step 2] 질문 들어오면 관련 조례만 검색 [Step 3] 그 조례를 기반으로 답변 생성

"Few-shot이랑 뭐가 달라요?"

"Few-shot은 예시 3~5개. RAG는 문서 전체를 넣는 거죠."

민준이 설명했다.

"나도 구산구 조례 전체를 RAG로 학습시켰어. 200페이지든 2,000페이지든 다 찾아내."

"대박..."

윤서연은 노트에 적었다. 형광펜 3색.

"그럼 대진시 조례 전체를 넣으면..."

"네. 자동으로 찾아서 답변해요."

최지훈은 노트북을 쳤다. 타닥타닥타닥.

"하지만 문제가 있어요."

"뭔데요?"

"예산이요."

"RAG 시스템 구축에 5천만 원 필요합니다."

"5천만 원?"

윤서연은 안경을 밀어 올렸다.

윤서연은 형광펜으로 칠한 조례를 펼쳤다.

"또 AI? 프롬프트 100개로 만족 못 해?"

탁. 정민수는 서류를 내려놓았다.

🎓 AI 튜토리얼: RAG로 방대한 문서 정복하기

💡 Logic (원리)

RAG (Retrieval-Augmented Generation)는 AI가 방대한 문서를 검색해서 답변하는 기술입니다.

핵심 원리:

1. 문서 임베딩: 200페이지 조례를 AI가 이해할 수 있는 형태로 변환

2. 검색 (Retrieval): 질문이 들어오면 관련 부분만 찾기

3. 생성 (Generation): 찾은 내용을 바탕으로 답변 생성

Few-shot vs RAG 비교:

Few-shot: 예시 3~5개 제공 (수동) RAG: 문서 전체 제공 (자동 검색)

실전 효과:

- 윤서연의 프롬프트 100개 → RAG 1개로 대체 가능

- 200페이지 외워야 할 조례 → AI가 자동 검색

- 매번 복사 붙여넣기 → 한 번 학습으로 끝

🛠️ Stack (도구)

기본 도구:

- ChatGPT Plus (유료) - GPT-4 필수

- 또는 Claude Projects (무료 가능)

고급 도구 (선택):

- LangChain (개발자용)

- Pinecone (벡터 DB)

- 하지만 코딩 없이도 가능!

📋 Guide (실행 단계)

방법 1: ChatGPT File Upload (가장 쉬움)

Step 1: 문서 준비

- PDF 또는 TXT 파일로 준비 - 예: "대진시_민원_조례_전체.pdf" (200페이지)

Step 2: ChatGPT에 업로드

1. ChatGPT 열기 2. 클립 아이콘 클릭 3. 파일 업로드 4. "이 문서를 바탕으로 답변해줘" 명령

Step 3: 질문하기

"외국인 배우자 출생신고에 필요한 서류는?" → ChatGPT가 200페이지 중 관련 부분만 찾아서 답변

방법 2: Claude Projects (무료 가능)

Step 1: Project 생성

1. Claude.ai 접속 2. "New Project" 클릭 3. 프로젝트명: "대진시 민원 시스템"

Step 2: 문서 업로드

1. Project 내 "Add Knowledge" 클릭 2. 조례 PDF 업로드 3. Claude가 자동으로 학습

Step 3: 대화

"프로젝트 문서를 참고해서 답변해줘. 외국인 배우자 출생신고 절차는?" → 조례에서 자동 검색 후 답변

🎯 Prompt (바로 쓰는 템플릿)

기본 RAG 프롬프트:

[업로드한 문서 파일명: 대진시_민원_조례.pdf] 너는 대진시 민원봉사과 담당자야. 위에 업로드한 조례 문서를 참고해서 답변해줘. 규칙: 1. 반드시 문서 내용만 참고할 것 2. 관련 조례 번호를 명시할 것 3. 문서에 없는 내용은 "문서에 없음"이라고 답할 것 질문: [여기에 질문 입력]

실전 활용 예시:

[업로드: 대진시_민원_조례_2024.pdf - 200페이지] 너는 대진시 민원봉사과 담당자야. 위 조례 문서를 바탕으로 답변해줘. 질문: 중국인 유학생이 체류지 변경을 신청했습니다. 필요한 서류와 절차를 단계별로 알려주세요. 답변 형식: 1. 관련 조례 조항 2. 필요 서류 목록 3. 처리 절차 4. 소요 기간

복잡한 질문 예시:

[업로드: 대진시_전체_조례.pdf - 500페이지] 너는 대진시 통합 민원 담당자야. 질문: 외국인 주민이 자녀 출생신고 + 건강보험 가입 + 주민등록을 한 번에 하고 싶어 합니다. 통합 처리가 가능한가요? 가능하다면 어떤 순서로 진행해야 하나요? 답변 형식: 1. 통합 처리 가능 여부 (조례 근거) 2. 순서별 절차 3. 각 단계별 필요 서류 4. 총 소요 기간

Tips:

- 파일은 PDF가 가장 안정적 (DOCX도 가능)

- 한 번에 10개 파일까지 업로드 가능 (ChatGPT Plus)

- 파일 크기는 최대 25MB

- 이미지 스캔본보다 텍스트 PDF가 정확도 높음

⚠️ 주의사항

RAG 한계:

1. 문서에 없는 내용은 답변 불가

2. 문서가 애매하면 답변도 애매

3. 최신 법령 개정은 수동 업데이트 필요

해결책:

- 정기적 문서 업데이트 (월 1회 권장)

- 애매한 조항은 Few-shot 예시 추가

- 중요한 답변은 반드시 사람이 최종 검토
