## 🎓 AI 튜토리얼: 템플릿 자동화로 시간 10배 절약

### 💡 Logic (원리)

**템플릿 자동화**는 반복되는 업무를 변수로 치환하는 기법입니다. Ep 2의 Few-shot이 "사례 학습"이었다면, 템플릿은 **"효율화"**입니다.

**핵심 원리:**
1. **반복 = 템플릿화 가능**: 같은 구조가 반복되면 변수로 바꿀 수 있습니다.
2. **변수 = 빈칸**: [이름], [날짜], [금액] 같은 빈칸을 채우는 방식
3. **1개 템플릿 = 무한 활용**: 한 번 만들면 계속 재사용

**실전 사례:**
- 윤서연의 프롬프트 100개 = 실제로는 10개 템플릿 × 10가지 변형
- 매번 처음부터 안 써도 됨 → 빈칸만 채움

### 🛠️ Stack (도구)

- **ChatGPT** (무료/유료)
- **메모장/노션** (템플릿 보관)
- **Ctrl+C, Ctrl+V** (복사 붙여넣기의 힘)

### 📋 Guide (실행 단계)

**Step 1: 반복 패턴 찾기**
```
예시) 민원 처리를 10번 하다 보니 구조가 똑같음:
- 고객 상황 설명
- 관련 조례 찾기
- 필요 서류 안내
- 처리 기간 알림
```

**Step 2: 변수 추출**
```
고정: 구조 (상황 → 조례 → 서류 → 기간)
변동: 내용 (민원 유형, 조례 번호, 서류 종류)

→ 변동하는 부분을 [변수]로 표시
```

**Step 3: 템플릿 제작**
```
너는 대진시 민원 담당자야.

상황: [민원_유형]
조례: [대진시_조례_번호]
필요서류: [서류_목록]
처리기간: [N]일

위 정보로 민원인에게 안내문을 작성해줘.
```

**Step 4: 빈칸 채워서 사용**
```
[민원_유형] → 전입신고
[대진시_조례_번호] → 제7조
[서류_목록] → 전입신고서, 신분증
[N] → 1

→ 완성된 프롬프트 생성
```

### 🎯 Prompt (바로 쓰는 템플릿)

**기본 템플릿 (민원 처리용):**

```
너는 [조직명] [부서] 담당자야.

고객 상황: [상황_설명]
관련 규정: [조례/법령_번호]
필요 서류: [서류1], [서류2], [서류3]
처리 기간: [N]일 소요
특이사항: [있으면_기재/없으면_생략]

위 내용을 바탕으로:
1. 고객에게 보낼 안내문 작성
2. 각 서류가 필요한 이유 설명
3. 처리 절차를 단계별로 정리

형식: 공문서 어조, 존댓말, 3문단 이내
```

**실전 활용 예시 1 (주차 위반):**

```
너는 대진시 민원봉사과 담당자야.

관련 규정: 대진시 조례 제12조, 도로교통법 제160조
필요 서류: CCTV 영상 확인 요청서
처리 기간: 3일 소요
특이사항: 영상 확인 결과에 따라 이의 인정/거부 결정

위 내용을 바탕으로:
1. 고객에게 보낼 안내문 작성
2. CCTV 확인이 필요한 이유 설명
3. 처리 절차를 단계별로 정리

형식: 공문서 어조, 존댓말, 3문단 이내
```

**실전 활용 예시 2 (외국인 민원):**

```
너는 대진시 민원봉사과 담당자야.

고객 상황: 중국인 유학생 비자 연장 서류 접수
관련 규정: 대진시 조례 제18조, 출입국관리법 제25조
필요 서류: 체류지 변경신고서, 여권 사본, 재학증명서, 통장사본
처리 기간: 7일 소요
특이사항: 영문 서류는 번역 공증 필요

위 내용을 바탕으로:
1. 고객에게 보낼 안내문 작성 (한글 + 영어)
2. 각 서류가 필요한 이유 설명
3. 처리 절차를 단계별로 정리

형식: 공문서 어조, 존댓말, 3문단 이내
```

**나만의 템플릿 만드는 법:**

1. 자주 하는 업무 3가지 선정
2. 각 업무의 공통 구조 파악
3. 변하는 부분을 [변수] 처리
4. 메모장에 저장 (파일명: "내업무_템플릿.txt")
5. 필요할 때마다 복붙 → 빈칸 채우기

**Tips:**
- 템플릿은 10개면 충분 (너무 많으면 관리 힘듦)
- 변수명은 직관적으로 ([날짜], [이름], [금액])
- 1달에 한 번 템플릿 업데이트 (규정 변경 반영)
